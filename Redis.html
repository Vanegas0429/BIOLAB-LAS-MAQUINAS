Características

  • Base de datos NoSQL en memoria, clave-valor.
  • Muy rápida para cachés y sesiones de usuario.
  • Soporta estructuras como listas, conjuntos y hashes.
  • Ideal para tiempo real y alta velocidad.

Ejemplos CRUD

# Insertar registro
SET usuario:1 "Juan Pérez"
HSET usuario:1 nombre "Juan Pérez" correo "juan@example.com"

# Crear tabla
SET usuario:1 "Pedro"

# Consultar
GET usuario:1

# Actualizar
SET usuario:1 "Pedro Gómez"

# Eliminar
DEL usuario:1
