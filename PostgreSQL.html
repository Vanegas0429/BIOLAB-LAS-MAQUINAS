Características

  • Base de datos relacional y objeto-relacional.
  • Muy robusta y estándar SQL.
  • Extensible con funciones personalizadas.
  • Excelente manejo de transacciones ACID.

Ejemplos CRUD

-- Insertar registro
INSERT INTO usuarios (nombre, correo)
VALUES ('Juan Pérez', 'juan@example.com')
RETURNING id;

-- Crear tabla
CREATE TABLE empleados (
  id SERIAL PRIMARY KEY,
  nombre VARCHAR(100),
  puesto VARCHAR(100)
);

-- Consultar
SELECT * FROM empleados;

-- Actualizar
UPDATE empleados SET puesto='Gerente' WHERE id=1;

-- Eliminar
DELETE FROM empleados WHERE id=1;
